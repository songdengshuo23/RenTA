## 5. 查询注册账户

不要查询或导出 `hashed_password`、`access_token`、`refresh_token`。

```sql
select
  u.id,
  u.username,
  u.phone,
  u.email,
  u.name,
  u.is_active,
  u.created_at,
  array_agg(r.name) as roles
from account_user u
left join account_user_role_link l on l.user_id = u.id
left join account_role r on r.id = l.role_id
group by u.id
order by u.created_at desc;
```

## 6. 查询平台广场可见 Agent

广场可见口径：已审核通过、启用、未删除、未禁用。

```sql
select
  a.id,
  a.aic,
  a.name,
  a.version,
  a.description,
  a.approval_status,
  a.is_active,
  a.is_deleted,
  a.is_disabled,
  u.username as created_by,
  a.created_at,
  a.processed_at
from agent a
left join account_user u on u.id = a.created_by_id
where a.approval_status = 'APPROVED'
  and a.is_active = true
  and a.is_deleted = false
  and a.is_disabled = false
order by a.processed_at desc nulls last, a.created_at desc;
```

### 11.9 按 AIC 查编排端可见性

把 `AGENT_AIC` 换成 Agent 的 `aic`：

```bash
AGENT_AIC='1.2.156.3088.0001.00001.K133L5.1OH7H0.1.03IO'
curl "http://127.0.0.1:8001/acps-atr-v2/passports/${AGENT_AIC}/dispatch" \
  -H 'Authorization: Bearer local-dev-token'
```

如果这里返回 `200` 且结果允许派发，说明编排端按 AIC 查询 Passport 准入是通的。

### 11.10 用 API 查本人提交后的审核结果

如果手上有登录后的 JWT，也可以不用直接查库：

```bash
AGENT_ID='019ed35b-0da7-7258-a46c-30b55828b5ff'
JWT='<登录返回的 access_token>'

curl "http://127.0.0.1:8001/api/agent/client/${AGENT_ID}/supervisor-review/latest" \
  -H "Authorization: Bearer ${JWT}"

curl "http://127.0.0.1:8001/api/agent/client/${AGENT_ID}/passport/latest" \
  -H "Authorization: Bearer ${JWT}"
```

这些 API 只允许创建者本人查询。管理员或工作人员要看全量列表，优先用：

```bash
curl 'http://127.0.0.1:8001/api/agent/staff?page_num=1&page_size=50&with_users=true' \
  -H "Authorization: Bearer ${JWT}"
```
